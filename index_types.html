Index of types


A
author [Syndic_atom]
Describes a person, corporation, or similar entity (hereafter, 'person') that indicates the author of the entry or feed.

B
body [Syndic_opml1]
List of outline elements.

C
category [Syndic_atom]
The category element conveys information about a category associated with an entry or feed.
category [Syndic_rss2]
category is an optional sub-element of Syndic_rss2.item.
channel [Syndic_rss2]
Here's a list of the required channel elements, each with a brief description, an example, and where available, a pointer to a more complete description.
channel [Syndic_rss1]
The channel element contains metadata describing the channel itself, including a title, brief description, and URL link to the described resource (the channel provider's home page, for instance).
channel_image [Syndic_rss1]
Establishes an RDF association between the optional image element 5.4 and this particular RSS channel.
channel_textinput [Syndic_rss1]
Establishes an RDF association between the optional textinput element 5.6 and this particular RSS channel.
cloud [Syndic_rss2]
cloud is an optional sub-element of Syndic_rss2.channel.
content [Syndic_atom]
content either contains or links to the content of the entry.

D
description [Syndic_rss1]
This can be a brief description of the channel's content, function, source, etc. See RSS 1.0 § 5.3.3;, or a brief description/abstract of the item. See RSS 1.0 § 5.5.3;, or a brief description of the textinput field's purpose. For example: "Subscribe to our newsletter for..." or "Search our site's archive of..." See RSS 1.0 § 5.6.2. 
Syntax: <description>{description}</description>

Requirement: Required only for channel and textinput

Model: (#PCDATA)

(Suggested) Maximum Length: 500 for channel and item

(Suggested) Maximum Length: 100 for textinput
dtd [Syndic_xml]
The type for the optional DTD.

E
enclosure [Syndic_rss2]
enclosure is an optional sub-element of Syndic_rss2.item.
entry [Syndic_atom]
entry represents an individual entry, acting as a container for metadata and data associated with the entry.
error [Syndic_w3c]

F
feed [Syndic_atom]
feed is the document (i.e., top-level) element of an Atom Feed Document, acting as a container for metadata and data associated with the feed.

G
generator [Syndic_atom]
The generator element's content identifies the agent used to generate a feed, for debugging and other purposes.
guid [Syndic_rss2]
guid is an optional sub-element of Syndic_rss2.item.

H
head [Syndic_opml1]

I
icon [Syndic_atom]
The icon element's content is an IRI reference RFC3987 that identifies an image that provides iconic visual identification for a feed.
id [Syndic_atom]
The id element conveys a permanent, universally unique identifier for an entry or feed.
image [Syndic_rss2]
An image is an optional sub-element of Syndic_rss2.channel, which contains three required (url, title, link) and three optional (width, height, description) sub-elements.
image [Syndic_rss1]
An image to be associated with an HTML rendering of the channel.
item [Syndic_rss2]
A Syndic_rss2.channel may contain any number of items.
item [Syndic_rss1]
While commonly a news headline, with RSS 1.0's modular extensibility, this can be just about anything: discussion posting, job listing, software patch -- any object with a URI.
items [Syndic_rss1]
An RDF table of contents, associating the document's items 5.5 with this particular RSS channel.

K
kind [Syndic_w3c]
Distinguishes an error from a warning.

L
link [Syndic_atom]
link defines a reference from an entry or feed to a Web resource.
link [Syndic_rss1]
This can be The URL to which an HTML rendering of the channel title will link, commonly the parent site's home or news page. See RSS 1.0 § 5.3.2, Or the URL to which an HTML rendering of the channel image will link. This, as with the channel's title link, is commonly the parent site's home or news page. See RSS 1.0 § 5.4.3, Or the item's URL. See RSS 1.0 § 5.5.2, Or the URL to which a textinput submission will be directed (using GET). See RSS 1.0 § 5.6.4 
Syntax: <link>{link}</link>

Requirement: Required for all

Model: (#PCDATA)

(Suggested) Maximum Length: 500
logo [Syndic_atom]
logo is an IRI reference RFC3987 that identifies an image that provides visual identification for a feed.

M
mime [Syndic_atom]
A MIME type that conform to the syntax of a MIME media type, but MUST NOT be a composite type (see Section 4.2.6 of MIMEREG).
month [Syndic_date]
Month of the year.

N
name [Syndic_rss1]
The text input field's (variable) name.

O
outline [Syndic_opml1]

P
published [Syndic_atom]
published is a Date construct indicating an instant in time associated with an event early in the life cycle of the entry.

R
rdf [Syndic_rss1]
The outermost level in every RSS 1.0 compliant document is the RDF element.
rel [Syndic_atom]
Indicates the link relation type.
rights [Syndic_atom]
rights is a Text construct that conveys information about rights held in and over an entry or feed.

S
source [Syndic_atom]
If an Syndic_atom.entry is copied from one feed into another feed, then the source Syndic_atom.feed's metadata (all child elements of atom:feed other than the atom:entry elements) MAY be preserved within the copied entry by adding an atom:source child element, if it is not already present in the entry, and including some or all of the source feed's Metadata elements as the atom:source element's children.
source [Syndic_rss2]
source is an optional sub-element of Syndic_rss2.item.
story [Syndic_rss2]
subtitle [Syndic_atom]
subtitle is a Text construct that conveys a human-readable description or subtitle for a feed.
summary [Syndic_atom]
summary is a Text construct that conveys a short summary, abstract, or excerpt of an entry.

T
t [Syndic_error]
t [Syndic_xml]
A XML tree.
t [Syndic_date]
A date with time.
t [Syndic_w3c]
t [Syndic_opml1]
text_construct [Syndic_atom]
A text construct.
textinput [Syndic_rss2]
A Syndic_rss2.channel may optionally contain a textInput sub-element, which contains four required sub-elements.
textinput [Syndic_rss1]
The textinput element affords a method for submitting form data to an arbitrary URL — usually located at the parent website.
title [Syndic_atom]
title is a Text construct that conveys a human-readable title for an entry or feed.
title [Syndic_rss1]
A descriptive title for the channel, image, item and textinput.

U
updated [Syndic_atom]
updated is a Date construct indicating the most recent instant in time when an entry or feed was modified in a way the publisher considers significant.
url [Syndic_rss1]
The URL of the image to used in the "src" attribute of the channel's image tag when rendered as HTML.

W
warning [Syndic_w3c]
